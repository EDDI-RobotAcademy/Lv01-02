# Homework

```make
1. 오늘 작성한 switch에 대해 어셈블리 분석을 해봅니다.
2. continue에 대한 어셈블리 분석을 합니다.
3. 일반적인 break 방식의 에러 핸들링과
   goto 방식의 에러 핸들링 코드를 어셈블리 분석 해봅니다.
4. 배열에 대한 어셈블리 분석을 수행합니다.
5. 오늘 학습한 포인터에 대한 어셈블리 분석을 통해
   결국 모든 방식이 포인터 형식으로 구성된다는 것을 이해하도록 합니다.

6. 3명의 사원이 있고 각각의 사원에게 2400 ~ 3600 사이의 랜덤한 급여를 지급합니다.
   1년마다 연봉 인상률을 1 ~ 10% 적용하여 10년후 각 사원들의 급여를 출력해보도록 합니다.
7. 컴퓨터와 주사위를 굴려서 숫자가 높은 사람이 이기도록 프로그래밍 해봅시다.
8. 7번 문제의 확장으로 주사위를 3개 굴려서 숫자가 높은 사람이 이기도록 만들어봅시다.
   (주사위 눈금을 모두 더한 결과를 비교한다)
9. 조금 더 확장해서 3명의 플레이어가 주사위를 가지고 게임을 한다 가정합니다.
   주사위를 2개 굴려서 숫자가 높은 사람이 이기게 해봅시다.
   (마찬가지로 주사위 눈금을 모두 더해 비교하도록 한다)
10. 9번 주사위 게임에 추가적인 옵션 기능을 구현합니다.
    1번째 주사위가 홀수면 주사위를 1개만 굴릴 수 있습니다.
    만약 짝수가 나오면 2번째 주사위를 굴릴 수 있습니다.
    2번째 주사위는 옵션이 있는데 숫자 4가 나오면 0점 처리됩니다.
    숫자 3이 나온 경우엔 1번째 굴린 주사위를 무조건 2배수 처리합니다.
    숫자 6이 나온 경우엔 자신을 제외한 모든 플레이어들의 주사위 숫자를 -4 처리합니다.
	숫자 1이 나오면 지정한 사람의 주사위를 +3 할 수 있고
    추가로 또 한 사람을 지명하여 -2 할 수 있습니다.
```
